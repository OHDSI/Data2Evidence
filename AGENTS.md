# AGENTS.md

## Overview

Data2Evidence (D2E) is a healthcare data analytics platform built on OHDSI standards and OMOP Common Data Model. It provides tools for clinical research, cohort analysis, and data quality assessment.

## Tech Stack

- **Runtime**: Trex (Rust gateway + Deno V8 for TypeScript)
- **Functions**: TypeScript/Deno with Danet framework
- **Flows**: Python 3.x with Prefect for data pipelines
- **Frontend**: React 18, Vue 3, SAP UI5
- **Database**: PostgreSQL (primary), DuckDB (analytics cache)
- **Build**: Bun, NX (monorepo), Docker Compose

## Commands

```bash
# Setup and initialization
npm run init                    # Generate .env.local config
npm run build                   # Build local services
npm run start                   # Start all services
npm run setup                   # Load demo OMOP CDM dataset

# Development
npm run logs                    # View service logs
npm run stop                    # Stop services
npm run config                  # Show current config
npm run ps                      # List running containers

# Database
npm run create-postgres-cdm-schemas  # Create OMOP CDM schemas

# Testing
npm run inittestdb              # Initialize test database
npm run removetestdb            # Clean test database
cd tests/backend_integration_tests && npm test
```

## Architecture

```
d2e/
├── functions/      # TypeScript/Deno microservices (Danet framework)
├── flows/          # Python/Prefect data pipelines
├── ui/             # Frontend apps (React, Vue 3, SAP UI5)
├── services/       # Infrastructure (trex, dataflow, cachedb)
├── cache/          # Shared data (vocab, test data, TLS certs)
└── deploy/         # Docker & deployment configs
```

### Service Communication

- All requests route through **Trex** gateway on port 41111
- Services discovered via environment variables (`SERVICE_HOST`, `SERVICE_PORT`)
- Internal TLS certificates in `/cache/tls/`

### Plugin System

- **Functions**: Registered in `functions/package.json` under `trex.functions`
- **UI**: Registered in `ui/package.json` under `trex.ui`
- **Flows**: Python plugins in `flows/` with `flow.py` entry point

## Code Style

### TypeScript/Deno Services

```typescript
// Use Danet decorators for controllers
@Controller('/api/resource')
export class ResourceController {
  constructor(private readonly service: ResourceService) {}

  @Get('/:id')
  async getById(@Param('id') id: string) {
    return this.service.findById(id);
  }
}

// Use TypeORM for database entities
@Entity('table_name')
export class MyEntity {
  @PrimaryGeneratedColumn()
  id: number;
}
```

## Environment

Key environment variables:
- `DATABASE_CREDENTIALS` - Multi-database connection JSON
- `PLUGINS_SEED` - Array of plugins to load
- `RUST_LOG` - Logging level (info, debug)

Config files:
- `.env.local` - Local development (generated by `npm run init`)
- `docker-compose-local.yml` - Local service orchestration

## Important Files

- `package.json` - Root scripts and version
- `functions/package.json` - Function registration (`trex.functions`)
- `ui/package.json` - UI routes and plugins (`trex.ui`)
- `deploy/caddy-config/Caddyfile` - Reverse proxy config
- `scripts/cli.sh` - Main D2E CLI implementation
